## 2. 项目概览与核心体验 (2-3分钟)
**[幻灯片：项目首页截图或功能架构图]**

首先，让我们来看看这个系统能做什么。我将其核心体验定义为三个维度：**安全便捷的准入**、**流畅的创作体验**以及**可控的互动管理**。

### 安全便捷的准入
**[幻灯片：登录/注册页面截图]**
在用户系统设计上，我首要考虑的是“安全”与“门槛”的平衡。
- **注册环节**：为了防止恶意机器注册，我实现了**邮箱验证码机制**。系统会通过 SMTP 服务向用户发送 6 位随机数字验证码，确保身份的真实性。
- **登录环节**：除了传统的账号密码登录，我还集成了 **Gitee OAuth 第三方登录**。用户无需记忆新密码，一键即可授权登录，极大地降低了使用门槛。

### (2) 流畅的创作体验
**[幻灯片：文章编辑与详情页截图]**
作为博客的核心，内容管理至关重要。
*   **前台界面与阅读体验**：
    *   **响应式设计**：前台采用了 **Bootstrap 5** 响应式布局，通过 CSS Media Queries 适配移动端和桌面端，无论是在手机还是电脑上，读者都能获得一致的流畅阅读体验。
    *   **沉浸式阅读**：文章详情页经过精心排版，优化了字体间距与行高，配合 **Thymeleaf** 的服务端渲染，首屏加载速度极快，最大程度减少读者的等待时间。
*   **富文本编辑**：系统集成了 **wangEditor** 富文本编辑器，支持图文混排，配合后端的 **LocalStorageProvider** 实现图片的本地化存储，让创作过程所见即所得。
*   **定时发布**：这里有一个特别的功能——**定时发布**。利用 Spring Boot 的 **@Scheduled** 注解实现后台任务调度，系统会每分钟自动扫描数据库中达到预定时间的草稿，将其状态原子性地更新为“已发布”。这对于保持更新频率非常有帮助。

### (3) 可控的互动管理
**[幻灯片：评论区与后台管理截图]**
为了维护良好的社区氛围，我引入了**评论审核机制**。
*   **后台界面与内容管控**：
    *   **仪表盘概览**：后台首页提供数据可视化仪表盘，管理员可实时监控文章总数、待审核评论数等关键指标。
    *   **集中化管理**：通过统一的后台管理界面，管理员可以对文章、分类、标签进行增删改查（CRUD）操作。特别是**评论管理模块**，支持一键通过或驳回，极大提升了内容管控的效率。
*   **权限控制**：基于 **RBAC（基于角色的访问控制）** 模型，普通访客提交的评论状态默认为“待审核”。
*   **技术实现**：管理员在后台通过 **Thymeleaf** 渲染的管理面板进行审核，只有状态被更新为“已批准”的评论才会在前台通过 **AJAX** 异步加载显示。这有效防止了垃圾信息的泛滥。

## 3. 技术架构与实现亮点 (3-4分钟)
**[幻灯片：技术栈概览 - Spring Boot, MySQL, JPA, Security]**

在简洁的界面背后，是一套坚实的技术架构在支撑系统的运行。我想重点介绍三个技术亮点：

### 第一：企业级的安全认证体系
**[幻灯片：JWT + Spring Security 流程图]**
安全性是 Web 应用的基石。
- 我采用了 **Spring Security** 配合 **JWT (JSON Web Token)** 的无状态认证方案。
- 当用户登录成功后，服务器签发加密的 Token，客户端将其存储在本地。之后的每一次请求，都会携带这个“数字身份证”。
- 这种设计不仅降低了服务器内存压力，还为未来扩展移动端 App 或前后端分离架构打下了基础。
- 同时，所有用户密码都经过 **BCrypt 哈希算法** 加密存储，即使数据库被攻破，黑客也无法还原出原始密码，最大程度保障用户信息安全。

### 第二：优雅的数据交互
**[幻灯片：Spring Data JPA 代码片段]**
在数据持久层，我使用了 **Spring Data JPA** 作为 ORM 框架。它充当了 Java 对象与 MySQL 数据库之间的“翻译官”，让我能以面向对象的方式操作数据库，极大地提高了开发效率和代码的可维护性。

### 第三：混合渲染模式
**[幻灯片：Thymeleaf + AJAX 示意图]**
在前端实现上，我采用了 **Thymeleaf 服务端渲染** 与 **原生 JavaScript 动态交互** 相结合的模式。
- 首页和文章页由服务端直接生成 HTML，这对搜索引擎（SEO）非常友好，利于文章被收录。
- 而点赞、评论等交互功能，则通过 AJAX 异步请求 API，保证了用户体验的流畅性，无需频繁刷新页面。
